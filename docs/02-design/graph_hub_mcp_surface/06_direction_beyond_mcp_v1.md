# 06 Direction Beyond MCP v1

- Status: Future direction, not v1 implementation scope
- Depends on: stable Phase 0-4 foundations

## Purpose

This document captures where FigOps should go after MCP v1 is stable. These items are intentionally downstream of discovery truth, read-only MCP, controlled rendering, project normalization, and visual quality gates.

## 1. Graph Inventory Board

Create a persistent, reviewable inventory of all graph-related projects and artifacts.

Fields:

- project classification,
- figure IDs and output paths,
- data contract status,
- style target,
- last run status,
- visual review status,
- migration readiness.

Rules:

- generated by an explicit inventory command or MCP tool,
- not generated by passive health checks,
- safe to inspect without executing project scripts,
- usable by humans and agents.

## 2. Adapter Registry

Support reusable adapters for common raw-data families only after the structured CSV path is stable.

Example adapter families:

- resistance time series,
- dielectric measurement tables,
- actuator displacement curves,
- cyclic mechanical test summaries.

Each adapter should declare:

- input signatures,
- output schema,
- validation checks,
- example fixtures,
- supported plot templates,
- failure modes.

Adapter work must not blur the line between raw data analysis and graph rendering. Raw analysis remains project-specific until an adapter is explicitly specified and tested.

## 3. Manuscript Figure Pipeline

Extend from single graph rendering to manuscript figure preparation.

Capabilities:

- figure pack creation,
- multi-panel assembly,
- journal-specific size checks,
- baseline visual regression,
- reviewer-ready artifact manifests.

This should remain downstream of reliable project discovery and visual verification.

## Out of Scope for MCP v1

- automatic analysis of all raw instrument files,
- universal graph type inference,
- manuscript-wide batch generation without project validation,
- replacing project-specific `hub_scripts/`,
- fully autonomous publication readiness claims.
